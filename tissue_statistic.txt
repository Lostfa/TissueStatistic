================================================================================
                    TissueStatistic — 方法与数据处理
================================================================================

【中文版本】

1. 研究工具概述

TissueStatistic是一个基于CT图像进行身体组织成分自动化定量分析的研究工具。该工具在BOA（Body and Organ Analysis）自动分割软件的基础上，构建了从图像标准化、组织分割到区域统计和结果整合的完整分析流程，可对大规模患者队列进行高效的组织成分量化评估。

2. CT图像预处理

为保障后续分割与统计分析的准确性，所有入组CT图像均进行标准化预处理。预处理支持DICOM及NIfTI两种原始格式的导入。具体步骤包括：（1）对CT值（Hounsfield Unit, HU）进行范围裁剪，以排除金属伪影及极端噪声的干扰；（2）沿纵轴（z轴）进行层间距重采样至1 mm，使体素接近各向同性，提升分割模型在冠状位和矢状位的重建精度，并为后续以毫米为单位的区域范围设定提供统一的空间尺度；（3）对于骨算法重建图像，可选应用高斯滤波（σ = 1.5）进行轻度平滑降噪。全部预处理结果以标准化格式统一存储，确保下游分析的输入一致性。

3. 基于深度学习的自动组织分割

标准化后的CT图像通过BOA软件进行全自动组织分割。BOA整合了两个已在CT图像分割领域得到广泛验证的深度学习模型：TotalSegmentator可一次性分割104个解剖结构，Body Composition Analysis（BCA）则专注于身体成分相关区域的识别。分割完成后，对每位受检者输出以下标签文件：（1）椎体分割标签——涵盖第2颈椎（C2）至第5腰椎（L5）共计23个椎体；（2）身体成分区域标签——包括骨骼、肌肉、皮下组织、腹腔、纵隔及心包等解剖分区；（3）组织成分标签——在BCA解剖分区的基础上，结合各组织的特征HU值范围，进一步将体素细分为7类目标组织，包括骨骼、肌肉、皮下脂肪（SAT）、腹腔脂肪（VAT）、肌间脂肪（IMAT）、纵隔脂肪（PAT）及心包脂肪（EAT）。

4. 组织成分统计分析方法

统计分析模块是本研究的核心方法学贡献，主要包含椎体定位、区域提取和定量统计三个环节。

4.1 椎体自动定位

基于BOA输出的椎体标签和BCA标签，对C2-L5范围内的23个椎体进行自动空间定位。为提升定位精度，首先利用BCA标签中的脊柱后部解剖标志对原始椎体标签进行清洗——将该标志尾侧的所有残余标签体素予以剔除，从而排除横突、棘突等非椎体主体结构对定位计算的干扰。随后通过计算目标椎体的空间质心，获得其在三维坐标系中的精确解剖定位，作为后续区域提取的空间参考基准。

4.2 区域提取策略

本工具提供两种互补的分析模式：（1）全身整体分析——对CT扫描范围内的全部层面进行组织成分统计，用于整体身体成分评估；（2）单椎体中心区域分析——以任意目标椎体的质心所在层面为中心，在设定的纵向范围内（可选1、5、10、20 mm及自定义范围）提取组织区域。后者的临床意义在于可针对特定脊柱节段进行局部组织成分的精细量化，例如评估L3椎体水平的腹部脂肪分布或T12水平的肌间脂肪浸润程度，已有研究表明这些脊柱标志层面的身体成分指标与代谢性疾病风险密切相关。所有区域提取均包含自动边界校验，防止超出图像有效范围。

4.3 组织类型与统计指标

对上述7种组织类型在每一分析区域内分别计算以下8项定量指标：体积（mm³）、HU最大值、HU最小值、HU均值、HU标准差、HU中位数、第25百分位数（Q1）及第75百分位数（Q3）。这些指标既反映组织的体量信息，又刻画其CT密度分布特征，可综合评估组织的“量”与“质”。

4.4 可自定义的HU阈值组织分类

考虑到不同扫描参数、重建算法和患者群体可能影响组织HU值的分布，本研究在BOA默认组织分类的基础上，额外提供了可自定义HU阈值的组织重分类功能。研究者可根据自身队列的成像条件和研究目的，灵活设定脂肪（默认-200~0 HU）和肌肉（默认1~150 HU）的HU鉴别范围，重新生成组织标签后再进行统计分析。这一设计增强了方法在不同设备和临床场景间的适用性和可比性。

5. 批量处理与数据整合

针对大规模队列研究的需要，工具采用并行处理架构，支持同时分析多名受检者。每位受检者的全部图像数据仅需加载一次，即可在单次运行中完成全身分析及全部23个椎体×4个预设范围（共计93项）的组织成分统计，显著减少了重复读写操作带来的时间开销。所有分析结果按受检者编号、椎体标识和范围参数的层级结构分别存储，便于后续筛选与合并。在分析完成后，工具可根据用户指定的组织类型和统计指标，自动生成面向统计软件（如SPSS、R等）直接导入的合并数据表格。

6. 操作流程

整个分析流程整合于一个支持浏览器访问的交互式操作界面，按以下4个步骤依次执行：（1）数据导入与预处理——上传CT图像，设定HU裁剪范围和可选滤波参数；（2）组织分割——选择分割模型并启动全自动分割；（3）统计分析——勾选目标椎体（按颈椎/胸椎/腰椎分组）和需要分析的范围（1-30 mm可调）；（4）结果导出——选择感兴趣的组织类型和统计指标，一键生成合并数据文件。操作界面右侧实时展示任务执行状态和日志信息，便于用户监控分析进度。

7. 相较于BOA的改进要点

（1）完整分析闭环：BOA仅完成组织分割步骤，本工具在其基础上实现了从图像预处理到统计报表生成的完整流程，避免了用户在多个独立工具间的数据传递和格式转换。
（2）椎体精确定位：引入基于BCA解剖标志的椎体标签清洗步骤，有效克服了原始椎体分割中非目标骨性结构的残留问题，大幅提升了椎体中心定位的解剖准确性。
（3）灵活的组织HU阈值设定：突破BOA固定阈值的限制，允许研究者依据不同设备和扫描协议调整组织分类的HU判别标准，提升了多中心研究中的方法一致性和结果可比性。
（4）脊柱节段特异性组织分析：以单个椎体为中心、可按毫米调节分析范围的设计，为特定脊柱水平的局部身体成分评估提供了标准化工具。
（5）面向队列研究的批量处理能力：通过单次加载多次分析的并行计算策略，实现了对大规模受检者队列的高效处理。
（6）交互式操作界面：将命令行工具转化为可视化操作流程，降低了使用门槛，并提供了灵活的结果筛选和合并导出功能。

             TissueStatistic — Methods and Data Processing
================================================================================

【English Version — for SCI Submission】

1. Overview

TissueStatistic is a dedicated research tool for automated quantitative analysis of body tissue composition from CT images. Building upon the BOA (Body and Organ Analysis) automatic segmentation software, this tool establishes a complete analytical workflow encompassing image standardization, tissue segmentation, region-specific statistical quantification, and result integration, enabling efficient body composition assessment across large patient cohorts.

2. CT Image Preprocessing

To ensure the accuracy of subsequent segmentation and statistical analyses, all enrolled CT images underwent standardized preprocessing. The preprocessing module accepts both DICOM and NIfTI input formats. Specific steps included: (1) clipping of Hounsfield Unit (HU) values to exclude interference from metal artifacts and extreme noise; (2) resampling of slice thickness to 1 mm along the longitudinal (z) axis to achieve near-isotropic voxels, thereby improving the reconstruction quality of the segmentation model in coronal and sagittal planes and providing a uniform spatial scale for the millimeter-based analysis range setting in subsequent steps; and (3) for images reconstructed with bone algorithms, optional mild Gaussian smoothing (σ = 1.5) was applied to reduce noise while preserving tissue boundaries. All preprocessed images were stored in a standardized format to ensure input consistency for downstream analyses.

3. Deep Learning-Based Automatic Tissue Segmentation

Standardized CT images were processed through the BOA software for fully automated tissue segmentation. BOA integrates two deep learning models that have been extensively validated in the domain of CT image segmentation: TotalSegmentator, which simultaneously delineates 104 anatomical structures, and Body Composition Analysis (BCA), which specifically focuses on the identification of body composition-related regions. Upon completion of segmentation, the following label files were generated for each subject: (1) vertebra segmentation labels — covering 23 vertebrae from the second cervical vertebra (C2) to the fifth lumbar vertebra (L5); (2) body composition region labels — encompassing skeletal, muscular, subcutaneous, abdominal, mediastinal, and pericardial anatomical compartments; and (3) tissue composition labels — derived by further classifying voxels into seven target tissue types based on characteristic HU ranges within the BCA anatomical partitions, namely BONE, MUSCLE, subcutaneous adipose tissue (SAT), visceral adipose tissue (VAT), intermuscular adipose tissue (IMAT), pericardial adipose tissue (PAT), and epicardial adipose tissue (EAT).

4. Tissue Composition Statistical Analysis

The statistical analysis module represents the core methodological contribution of this study, comprising three principal components: vertebra localization, region extraction, and quantitative statistics.

4.1 Automated Vertebra Localization

Based on the vertebra labels and BCA labels generated by BOA, automated spatial localization was performed for all 23 vertebrae from C2 to L5. To improve localization accuracy, the original vertebra labels were first refined using the posterior spinal anatomical landmarks from the BCA labels — residual label voxels inferior to this landmark were removed, thereby excluding non-vertebral-body structures such as transverse and spinous processes that would otherwise interfere with centroid computation. The spatial centroid of each target vertebra was subsequently calculated in three-dimensional coordinates, serving as a precise anatomical reference for the ensuing region extraction.

4.2 Region Extraction Strategy

This tool provides two complementary analysis modes: (1) whole-body analysis — tissue composition statistics are computed across all slices within the CT scan range, suitable for overall body composition assessment; and (2) vertebra-centered regional analysis — a tissue region is extracted within a user-defined longitudinal range (1, 5, 10, 20 mm, or custom values) centered on the slice containing the centroid of any target vertebra. The clinical significance of the latter lies in its capacity for fine-grained quantification of local tissue composition at specific spinal segments — for instance, evaluating abdominal adipose tissue distribution at the L3 vertebral level or intermuscular adipose infiltration at the T12 level, where body composition indices have been shown to correlate closely with metabolic disease risk. Automatic boundary verification is applied to all extracted regions to ensure they remain within the valid image domain.

4.3 Tissue Types and Statistical Metrics

For each of the seven tissue types within each analyzed region, the following eight quantitative metrics were computed: volume (mm³), maximum HU, minimum HU, mean HU, standard deviation, median HU, 25th percentile (Q1), and 75th percentile (Q3). These metrics capture both the volumetric extent and the CT attenuation distribution characteristics of each tissue, thereby enabling a comprehensive assessment of both tissue "quantity" and "quality."

4.4 Customizable HU Thresholds for Tissue Classification

Recognizing that variations in scanning parameters, reconstruction algorithms, and patient populations may influence the HU distribution of tissues, this study introduced a customizable HU threshold reclassification function beyond the default BOA tissue classification. Researchers may flexibly adjust the discriminative HU ranges for adipose tissue (default −200 to 0 HU) and muscle tissue (default 1 to 150 HU) according to the imaging conditions and study objectives of their own cohorts, with tissue labels regenerated prior to statistical analysis. This design enhances the applicability and comparability of the method across different scanner platforms and clinical settings.

5. Batch Processing and Data Integration

To accommodate the demands of large-scale cohort studies, the tool employs a parallel processing architecture capable of simultaneously analyzing multiple subjects. All image data for each subject are loaded only once, enabling completion of whole-body analysis and all 23 vertebrae × 4 preset ranges (a total of 93 analytical tasks) within a single run, substantially reducing the time overhead associated with repetitive data input/output operations. All analytical results are stored in a hierarchical structure organized by subject identifier, vertebra label, and range parameter, facilitating subsequent filtering and merging. Upon completion of the analysis, the tool can automatically generate merged data tables formatted for direct import into statistical software packages (e.g., SPSS, R).

6. Workflow

The entire analytical workflow is integrated into a browser-accessible interactive interface, executed sequentially across four steps: (1) Data import and preprocessing — uploading CT images, configuring the HU clipping range and optional filtering parameters; (2) Tissue segmentation — selecting segmentation models and initiating fully automated segmentation; (3) Statistical analysis — specifying target vertebrae (grouped by cervical/thoracic/lumbar regions) and analysis ranges (adjustable 1–30 mm); and (4) Result export — selecting tissue types and statistical metrics of interest, with one-click generation of a merged data file. The interface displays real-time task execution status and log information for user monitoring of analysis progress.

7. Key Improvements over BOA

(1) Complete analytical pipeline: Whereas BOA only performs the tissue segmentation step, the present tool achieves a full workflow from image preprocessing to statistical report generation, eliminating the need for data transfer and format conversion across multiple standalone tools.

(2) Precise vertebra localization: The introduction of a vertebra label refinement step based on BCA anatomical landmarks effectively addresses the issue of residual non-target osseous structures in the original vertebra segmentation, substantially improving the anatomical accuracy of vertebra centroid localization.

(3) Flexible HU threshold configuration: By overcoming the fixed-threshold limitation of BOA, this tool allows investigators to adapt the HU discriminative criteria for tissue classification according to different scanner platforms and acquisition protocols, thereby enhancing methodological consistency and cross-study comparability in multicenter research.

(4) Spine segment-specific tissue analysis: The design of vertebra-centered analysis with millimeter-adjustable range provides a standardized tool for localized body composition assessment at specific spinal levels.

(5) Batch processing capacity for cohort studies: The single-load multiple-analysis parallel computation strategy enables efficient processing of large subject cohorts.

(6) Interactive user interface: The transformation of command-line tools into a visual workflow lowers the usability barrier and provides flexible result filtering, merging, and export functionality.
